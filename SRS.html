1.Introduction

1.1 Purpose

We are aiming to build a digitized document management software which will change the entire documenting and file storage system to digital format for the department of finance. It will be accessible via web browser as well as phone application for searching and retrieving files.


1.2 Document Conventions

Document Conventions Use
OCR Scanning paper documents
DBMS Managing, storing, retrieving data
API User interface allows applicants to interact
ER User Hierarchy must have relation to other users and can take control of their actions
SOA communicate across different platforms and languages
Dashboard Dashboard for private and public folders, uploaded documents and real-time notification Viewer
CVS Format Upload and automatically link the attributes in designated file names without any limit
Django Framework
DMS Track, manage, store document and reduce paper

1.3 Intended Audience and Reading Suggestions

This project is prototype for the digitized document management system and it is limited to Finance department of Plan International Bangladesh. We are going to implement it under the supervision of Dr. Nabeel Mohammed and after the compilation of the software entire organization will be benefitted from it


1.4 Product Scope

The system will be user friendly and easily accessible, store, manage, edit, send and delete documents. It is based on web-application and Relational Database. Database server will be there for handling huge data form different city branches of the organization. As everything is digitized so it will be less time consuming and efficient than typical existing manual system. We vision to provide best possible software system with low pricing and easy-to-use application

2. Overall Description

2.1 Product Perspective

The product is a self-contained product which digitizes existing documents/files from the Department of Finance. The converted digital format is stored into storage defined by the Department of finance using a simple document system capable search and retrieval for attaching the same.The system will allow users to create accounts, upload unlimited files and search for required files. The system will contain helplines for users.The admin will be able to controll access,manage accounts, retrieve and recover data and reset password.

  1. Records Management

    • Dashboard
    • Unlimited Folders, User Groups and Lock System
    • Public and Private Repository
    • Uploading Documents
    • Unlimited Indexing
    • Uploading Index file in CSV format
    • Routing Records
    • Automatic Archive

  2. Accounts Management

    • Unlimited User Accounts, Profile picture Feature , User Management To Create New User and Link of User Account Profile
    • Password Reset Tool
    • Management Groups to Manage Folders
    • Google Login

  3. Search and Retrieval Management

    • Search Function and Filter Results
    • Optical Character Recognition (OCR)

  4. Reports Management

    • Records Count

  5. Notification Management

    • Email Notification
    • View Real-Time Notification

  6. System Management

    • Accessible in Web Browser,Supports Multiple Platforms,Viewable from Desktop, Tablet and Mobile
    • Service Oriented Architecture (SOA)
    • Policy for Brute Force Attacks

  7. Backup and Recovery

  8. Maintenance and Technical Support

    • Telephone Support / Helpdesk Facility 24 x 7 days

3. External Interface Requirements

3.1 User Interfaces

The user interface for the software shall be compatible to any browser such as Internet Explorer, Mozilla, or Netscape Navigator by which the user can access to the system. It can also access by Android Device through Android App. The user interface shall be implemented using any tool or software package like python, Django, Java, or kotlin, Android Studio.


3.2 Hardware Interfaces

Since the application must run over the internet, all the hardware shall require to connect internet will be hardware interface for the system. As for e.g. Modem, WAN – LAN, Ethernet Cross-Cable.


3.3 Software Interfaces

  1. The system should have a dashboard for private and publicfolders, uploaded documents and notifications.
  2. The system should have a public repository where users canonly view the uploaded/created record in assigned public folder.
  3. The system should have a private repository where onlypermitted users and groups are allowed to view and edit a record.
  4. The system should have lock feature tor folder/files.
  5. The system should have a group management to create ncw groups with specific roles per folder.
  6. The system should be capable to use Google Login or Active Directory as login authentication.
  7. The system should be capable to search data such as index, file name, date, author name, uploader name,
  8. The system should have a filtering in search result to easily track the documents.
  9. The system should be capable to count all the uploaded records by folder or user.
  10. The system should be capable to count all the pages in every record uploaded.

3.4 Communications Interfaces

This project supports all types of web browsers and android devices and the database is same. The system shall use the HTTP protocol for communication over the internet and for the intranet communication will be through TCP/IP protocol suite




4. System Features

4.1 Description and Priority

The system should be stored into storage defined by the Department of finance using a simple document system capable search and retrieval for attaching the same.The project is high priority because it is very difficult to find the file in manually. This will make the work easy and time saving for the of department of finance


4.2 Stimulus/Response Sequences

  • A dashboard for private and public folders, uploaded documents and notifications.
  • Users can Upload files in private and public repositories.
  • Users can search the dedicated file.
  • Users will get notifications through if there is any update on their section

REQ-1 :The system should have only one database where various platform user can access.

REQ-2:This is in a distributed system in which.

  • There are two site server sites and client sites where client can see the file and upload
  • In server site all things are managed by backed engineers
  • All the data resides at the server sites
  • All applications execute at the client sites


5. Other Nonfunctional Requirements

5.1 Performance Requirements

The system will be interactive and the delays involved will be less.So in every action-response of the system, there are no immediate delays. In case of opening windows ,forms of popping error messages and saving the settings or sessions there is delay much below 2 seconds, In case of opening databases, sorting questions and evaluation there are no delays and the operation is performed in less than 2 seconds for opening ,sorting, computing, posting the files. The performance of the system is dependent on how the load is delivered to the system.Due to increase workload , a peak workload will be defined.


5.2 Safety Requirements

System failure, login issues, account creation and password reset failure will cause damage or loss to the system. A backup system will be there for system failure which will be in archival storage.Information transmission should be securely transmitted to server without any changes in information.


5.3 Security Requirements

Security requirements includes authentication, authorization, data protection and so on . Authentication ensures that each entity involved in using a the service—the requestor, the provider, and the broker —is what it actually claims to be. Authorization determines whether the service provider has granted access to the Web service to the requestor. Data protection ensures that the Web servicerequest and response have not been tampered with en route. The main security concern is for users account hence proper login mechanism should be used to avoid hacking. The tablet id registration is way to spam check for increasing the security. Hence, security is provided from unwanted use of recognition software.


5.4 Software Quality Attributes

  • Availability

    • The System should be available on the specified date and specified time when many are required to use the product.

  • Correctness

    • The important aspects to consider in some environments are the professionalism and effectiveness of the website that could be measured through how many different platforms are supported and it supports adaptivity and adaptability for a personalization

  • Maintainability

    • Admins will maintain the ease and speed with which the system can be restored to operational status after a failure occurs

  • Usability

    • The system will be able to satisfy a maximum number of customers needs

  • Reliabity

    • The system will be aware of bias,look for sites with expertise, check the date ,avoid anonymous authors and check links


5.5 Business Rules

The system will include privacy policies,cookie consent notices , plagiarism and copyright laws , terms and conditions and disclaimer.




6. Other Requirements

  • System is flexible with resolution, to decrease file size the user can scan the documents in lower resolution
  • System will be allowed to scan the document in various formats (for example A4, A3, letter etc)
  • System can store the images scanned in various formats (for example PDF, DOC etc)
  • System will be maintained from time to time.
  • System will have backup, so that any document lost due to system failure can be recovered.
  • System will include indexes to the scanned images and the primary goal of indexing is to be able to easily search and extract data from the scanned images. This will also help the office's efficiencies by encouraging staff to browse for information without having to physically comb through boxes of files.

Appendix A: Glossary

  • OCR: Optical Character Recognition converts documents into editable and searchable data.
  • DMS: Documents management software stores, manage and track electronic documents and electronic images of paper-based information captured through the use of a document scanner.
  • DBMS: Database management system will store, retrieve, define, and manage data in a database.
  • API: Application programming Interface is a software intermediary that allows two applications to talk to each other.
  • SOA: Software Oriented Architecture makes it possible for adding new features and services without having to create them from scratch.
  • HTML: Hypertext Markup Language for documents designed to be displayed in a web browser.
